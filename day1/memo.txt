*자바스크립트 이론* 
프로그래밍: 사람이 원하는대로 컴퓨터가 작동할 수 있도록 컴퓨터 언어로 명령어를 나열하는 행위이다.
웹프로그래밍: 웹 브라우저를 동작시키는 프로그램을 작성하는 것
프렌트엔드 프로그래밍: 서버에서 받아온 정보를 웹 브라우저에 어떻게 표시할 것인지 프로그래밍


<자바스크립트의 특징>
1. 모든 웹 브라우저에서 작동한다.
2. 웹 브라우저에서 실행 결과를 즉시 확인할 수 있다.
3. 풀스택 웹 개발 뿐 아니라 다양한 용도의 프로그램을 만들 수 있다.
4. 다양한 자바스크립트 공개 API를 사용할 수 있다.
5. 다영한 라이브러리와 프레임워크를 사용할 수 있다.
                      ㄴ> 프레임워크에서 기본으로 제공하는 소스를
                          수정하거나 추가하는 방법으로 웹 프로그램
                          을 만들 수 있게 미리 준비한 일종의 틀이다.

  <코드 작성 규칙>
  1.세미콜론 : 하나의 명령어 끝날 때 세미콜론을 불여도 되고 붙이지 않아도 된다.
    ㄴ> 단, 한줄은 자바스크립트 엔진이 자동으로 세미콜론을 붙여주기 때문에 실행이 되지만,
        일부 명령에서는 가끔 에러가 발생이 되기 때문에 에러방지하기 위해서 세미클론을 
        붙여서 쓰는것을 권장합니다.

  2. 주석 : 주석은 코드에 관한 제사한 설명을 작성하거나 특정 코드를 임시로 사용하지 않게 할 때 사용합니다.
     한줄주석: 주석이 한줄일 때는 주석에 //를 넣으면 된다.
     여러 줄 주석: 한줄로 작성하기에는 주석이 너무 길 떄 여러 줄 주석을 사용합니다. 주석처리 하고싶은
                  부분을 /*와*/로 감싸면 됩니다.
  3. 들여쓰기: 1칸이나 3칸으로 들여쓰기를 해도 아무런 문제가 없지만, 다만 그분이 다른 의미가 있는 부분
              이라는것을 다른사람에게 알려줄 수 있어서 코드의 가독성이 향상됩니다.
              (*들여쓰기는 규칙적으로 하기를 권장)

 <간단한 프로그램을 실행 하기>
   console.log('Hello world');
    ㄴ> 출력하기전에 반드시 문자열은 항상 ''을 넣어야 실행이 가능하다.
